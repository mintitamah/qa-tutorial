🤖 บทที่ 3: Robot Framework

บทนี้จะสอน Robot Framework ตั้งแต่แนวคิด โครงสร้าง จนถึงเขียน test จริง

🤖 Robot Framework คืออะไร? มันคือ Tool ตัวไหน?

Robot Framework คือ tool สำหรับเขียน automated test ที่ใช้ภาษาอ่านง่าย เรียกว่า keyword-driven ใครก็อ่านรู้เรื่อง แม้ไม่รู้ code

ส่วนประกอบคืออะไรเปรียบเหมือน
VS Codeโปรแกรมที่ใช้เขียน codeโรงงาน (ที่ทำงาน)
Pythonภาษาที่ใช้ติดตั้ง libraryภาษาที่ใช้สั่งงาน
Robot Frameworktool ที่ใช้เขียน testเครื่องจักรในโรงงาน
Library (Browser/Requests)ชุดคำสั่งสำเร็จรูปที่ import มาใช้อุปกรณ์เสริมของเครื่องจักร
Robot Framework ต่างจาก Playwright ยังไง?

Playwright → เขียนด้วย Python ต้องรู้ programming
Robot Framework → เขียนด้วย keyword ภาษาธรรมดา ไม่ต้องรู้ programming ก็อ่านออก

ใช้ tool เดียวกันได้มั้ย? ได้! Robot Framework ใช้ Playwright เป็น engine ข้างใน ผ่าน Browser Library

🔑 Keyword-driven Testing คืออะไร?

แทนที่จะเขียน code ยาวๆ เราใช้ "keyword" ที่อ่านแล้วเข้าใจทันที

# Playwright — ต้องรู้ Python ถึงจะอ่านออก
page.fill("#user-name", "standard_user")
page.fill("#password", "secret_sauce")
page.click("#login-button")

# Robot Framework — อ่านเข้าใจทันทีแม้ไม่รู้ code
Fill Text    id=user-name    standard_user
Fill Text    id=password     secret_sauce
Click        id=login-button
ข้อดีของ Robot Framework:
✅ อ่านง่าย คนที่ไม่เขียน code ก็อ่านรู้เรื่อง
✅ report สวยงามในตัว ไม่ต้องติดตั้งเพิ่ม
✅ เหมาะกับทีมที่มีทั้ง dev และ non-dev
✅ ใช้ Playwright เป็น engine ข้างใน ผ่าน Browser Library

📐 โครงสร้าง Robot Framework มี 4 ส่วน — ทั้งหมดอยู่ในไฟล์เดียว

นี่คือความต่างจาก Playwright ที่แยกหลายไฟล์ Robot Framework ใส่ทุกอย่างในไฟล์ .robot ไฟล์เดียว

ส่วนหน้าที่ต้องมีมั้ย
*** Settings ***บอกว่าจะใช้ Library อะไร เหมือน import ใน Pythonถ้าใช้ Library ต้องมี
*** Variables ***ตัวแปรที่ใช้ซ้ำบ่อยๆ เก็บไว้ที่เดียวไม่บังคับ
*** Test Cases ***test case จริงๆ อยู่ตรงนี้ต้องมีเสมอ
*** Keywords ***สร้าง keyword ของตัวเองเพื่อใช้ซ้ำ เหมือน function ใน Pythonไม่บังคับ
Robot Framework — ทุกอย่างในไฟล์เดียว
─────────────────────────────────────
login.robot
├── *** Settings ***    ← import Library ที่จะใช้
├── *** Variables ***   ← ตัวแปรที่ใช้ซ้ำ เช่น URL, username
├── *** Test Cases ***  ← test จริงๆ แต่ละ test case อยู่ตรงนี้
└── *** Keywords ***    ← keyword ที่สร้างเองเพื่อใช้ซ้ำ


Playwright — แยกออกเป็นหลายไฟล์
─────────────────────────────────
โปรเจกต์/
├── pages/login_page.py   ← แผนที่หน้าเว็บ
└── tests/test_login.py   ← test case

📚 Library คืออะไร? มีอะไรบ้าง?

Library คือชุดของ keyword สำเร็จรูปที่เราเรียกใช้ได้เลย เหมือน import ใน Python

Libraryใช้ทำอะไรติดตั้งยังไง
Browserควบคุม browser (ใช้ Playwright เป็น engine) — ทำ UI testpip install robotframework-browser
RequestsLibraryยิง API เหมือน requests ใน Python — ทำ API testpip install robotframework-requests
BuiltInkeyword พื้นฐาน เช่น Log, Should Be Equal, Sleepมาพร้อม Robot ไม่ต้องติดตั้ง
Collectionsจัดการ list และ dictionaryมาพร้อม Robot ไม่ต้องติดตั้ง
Stringจัดการ text เช่น ตัดคำ แปลงตัวอักษรมาพร้อม Robot ไม่ต้องติดตั้ง
เปรียบกับ Playwright:
Robot: Library Browser → Playwright: from playwright.sync_api import Page, expect
Robot: Library RequestsLibrary → Playwright: import requests

ประเภทที่ 1: Built-in Library — มาพร้อม Robot ไม่ต้องติดตั้ง

Libraryใช้ทำอะไรตัวอย่าง keyword
BuiltInkeyword พื้นฐานทั่วไปLog, Should Be Equal, Sleep
Collectionsจัดการ list และ dictionaryGet From List, Dictionary Should Contain Key
Stringจัดการ textConvert To Uppercase, Split String
DateTimeจัดการวันเวลาGet Current Date, Add Time To Date
OperatingSystemจัดการไฟล์และโฟลเดอร์File Should Exist, Create Directory

ประเภทที่ 2: External Library — ต้องติดตั้งเพิ่ม

Libraryใช้ทำอะไรติดตั้ง
Browser LibraryUI test ใช้ Playwright เป็น engine — ใหม่กว่า แนะนำpip install robotframework-browser
SeleniumLibraryUI test ใช้ Selenium เป็น engine — เก่ากว่าpip install robotframework-selenium2library
RequestsLibraryAPI test ยิง HTTP requestpip install robotframework-requests
DatabaseLibraryต่อ database โดยตรงpip install robotframework-databaselibrary
AppiumLibrarytest mobile apppip install robotframework-appiumlibrary
SSHLibraryต่อ server ผ่าน SSHpip install robotframework-sshlibrary

🆚 Selenium คืออะไร? ต่างจาก Browser Library ยังไง?

นี่คือจุดที่หลายคนสับสน ต้องแยกให้ออกนะ

SeleniumPlaywright
คืออะไรtool/engine สำหรับควบคุม browsertool/engine สำหรับควบคุม browser
อายุเก่ากว่า มีมานานแล้วใหม่กว่า Microsoft สร้าง
ความเร็วช้ากว่า✅ เร็วกว่ามาก
Auto-wait❌ ต้องเขียน sleep() เอง✅ รอ element พร้อมเอง
ใช้ใน Robot ผ่านSeleniumLibraryBrowser Library
เปรียบให้เห็นภาพ:

Browser Library   ใช้  Playwright  เป็น engine  ← ใหม่กว่า เร็วกว่า แนะนำ
SeleniumLibrary   ใช้  Selenium    เป็น engine  ← เก่ากว่า ช้ากว่า

ทั้งคู่ทำสิ่งเดียวกัน คือควบคุม browser
แต่ Browser Library ดีกว่าในแทบทุกด้าน
แล้วทำไมบางบริษัทยังใช้ SeleniumLibrary อยู่?
เพราะมี codebase เก่าที่เขียนด้วย Selenium ไว้แล้ว การเปลี่ยนทั้งหมดต้องใช้เวลาและ effort มาก ถ้าเริ่มโปรเจกต์ใหม่ แนะนำให้ใช้ Browser Library เลย

🏆 โดยปกตินิยมใช้ Library ตัวไหน?

ประเภท testLibrary ที่นิยมเหตุผล
UI Automation (ใหม่)✅ Browser Libraryใช้ Playwright เป็น engine เร็วกว่า auto-wait
UI Automation (เก่า)SeleniumLibraryมี codebase เก่าอยู่แล้ว
API Automation✅ RequestsLibraryตัวเดียวที่นิยม ใช้งานง่าย
Mobile AutomationAppiumLibraryรองรับ iOS และ Android
สรุปจำง่ายๆ:

ทำ UI test → ใช้ Browser Library (ถ้าเริ่มใหม่)
ทำ UI test → ใช้ SeleniumLibrary (ถ้ามี codebase เก่า)
ทำ API test → ใช้ RequestsLibrary (ตัวเดียวที่นิยม)

⌨️ Keyword สำคัญที่ต้องรู้ใน Browser Library

Browser Library มี keyword ของตัวเองแบ่งเป็น 3 กลุ่มเหมือน Playwright

กลุ่มที่ 1: Navigation — เปิดเว็บ

New Browser    chromium    headless=true   # เปิด browser ใหม่
New Page       https://www.saucedemo.com   # เปิด tab ใหม่ไปที่ URL
Go To          https://www.saucedemo.com   # ไปที่ URL
Close Browser                              # ปิด browser

กลุ่มที่ 2: Action — ทำอะไรบนหน้าเว็บ

Fill Text      id=user-name    standard_user   # กรอกข้อความ
Click          id=login-button                 # กดปุ่ม
Select Options By    id=dropdown    value      # เลือก dropdown
Check Checkbox       id=checkbox               # ติ๊ก checkbox
Take Screenshot                               # ถ่ายรูปหน้าจอ
Hover          id=menu                        # เอาเมาส์ชี้ที่ element

กลุ่มที่ 3: Assertion — เช็คผล

Get Url        ==    https://saucedemo.com/inventory.html   # เช็ค URL
Get Text       id=error    contains    Username is required  # เช็คข้อความ
Get Element    id=login-button                              # เช็คว่า element มีอยู่มั้ย

🚀 เริ่มต้น Step 0 ถึงจบ — UI Automation

0 ติดตั้ง VS Code และ Python ตามที่อธิบายด้านบน
1 ติดตั้ง library 
pip install robotframework robotframework-browser
rfbrowser init
2 เปิดเว็บที่จะทดสอบ → คลิกขวา Inspect → หา id ของ element 
เปิด https://www.saucedemo.com
คลิกขวาที่ช่อง username → Inspect → id="user-name"
คลิกขวาที่ช่อง password → Inspect → id="password"
คลิกขวาที่ปุ่ม login   → Inspect → id="login-button"
3 สร้างไฟล์ robot_tests/login.robot 
# ===== ส่วนที่ 1: Settings =====
*** Settings ***
Library    Browser    # import Browser Library เพื่อควบคุม browser


# ===== ส่วนที่ 2: Variables =====
# เก็บค่าที่ใช้ซ้ำไว้ที่เดียว ถ้าต้องเปลี่ยนแก้แค่นี่
*** Variables ***
${URL}          https://www.saucedemo.com
${VALID_USER}   standard_user
${VALID_PASS}   secret_sauce


# ===== ส่วนที่ 3: Test Cases =====
*** Test Cases ***

Login With Valid Credentials
    Open Login Page                              # เรียก keyword ที่สร้างไว้ด้านล่าง
    Fill Text    id=user-name    ${VALID_USER}   # กรอก username
    Fill Text    id=password     ${VALID_PASS}   # กรอก password
    Click        id=login-button                 # กดปุ่ม login
    Get Url      ==    ${URL}/inventory.html     # เช็คว่า URL เปลี่ยนมั้ย

Login With Wrong Password
    Open Login Page
    Fill Text    id=user-name    ${VALID_USER}
    Fill Text    id=password     wrongpassword
    Click        id=login-button
    Get Text     [data-test=error]    contains    Username and password do not match

Login With Empty Username
    Open Login Page
    Fill Text    id=user-name    ${EMPTY}        # ${EMPTY} = ค่าว่าง
    Fill Text    id=password     ${VALID_PASS}
    Click        id=login-button
    Get Text     [data-test=error]    contains    Username is required

Login With Empty Password
    Open Login Page
    Fill Text    id=user-name    ${VALID_USER}
    Fill Text    id=password     ${EMPTY}
    Click        id=login-button
    Get Text     [data-test=error]    contains    Password is required

Login With Locked User
    Open Login Page
    Fill Text    id=user-name    locked_out_user
    Fill Text    id=password     ${VALID_PASS}
    Click        id=login-button
    Get Text     [data-test=error]    contains    Sorry, this user has been locked out


# ===== ส่วนที่ 4: Keywords =====
# สร้าง keyword เองเพื่อใช้ซ้ำ ไม่ต้องเขียนซ้ำทุก test
*** Keywords ***

Open Login Page
    New Browser    chromium    headless=true   # เปิด browser
    New Page       ${URL}                      # เปิดหน้าเว็บ
4 รัน test ใน Terminal 
# รัน test ทั้งหมด
robot robot_tests/login.robot

# รัน test เฉพาะ test case ที่ต้องการ
robot --test "Login With Valid Credentials" robot_tests/login.robot

# รันแล้วเก็บ report ไว้ในโฟลเดอร์ results
robot --outputdir results robot_tests/login.robot
5 ดู Report ที่ได้อัตโนมัติ 
report.html  → สรุปผล pass/fail ของทุก test case
log.html     → รายละเอียดทุก step ที่รัน พร้อม screenshot
output.xml   → ข้อมูลดิบสำหรับ CI/CD
6 Push ขึ้น CI/CD 
git add .
git commit -m "feat: add login Robot tests"
git push

🚀 เริ่มต้น Step 0 ถึงจบ — API Automation

Robot Framework ทำ API test ได้ด้วย โดยใช้ RequestsLibrary

1 ติดตั้ง RequestsLibrary 
pip install robotframework-requests
2 สร้างไฟล์ robot_tests/api_test.robot 
# ===== ส่วนที่ 1: Settings =====
*** Settings ***
Library    RequestsLibrary   # import library สำหรับยิง API


# ===== ส่วนที่ 2: Variables =====
*** Variables ***
${BASE_URL}    https://jsonplaceholder.typicode.com


# ===== ส่วนที่ 3: Test Cases =====
*** Test Cases ***

Get User Returns 200
    # สร้าง session เพื่อเชื่อมต่อกับ API
    Create Session    myapi    ${BASE_URL}

    # ส่ง GET request ไปที่ /users/1
    ${response}=    GET On Session    myapi    /users/1

    # เช็ค status code
    Should Be Equal As Numbers    ${response.status_code}    200

Get User Has Name Field
    Create Session    myapi    ${BASE_URL}
    ${response}=    GET On Session    myapi    /users/1

    # แปลง response เป็น dictionary
    ${body}=    Set Variable    ${response.json()}

    # เช็คว่ามี field name มั้ย
    Dictionary Should Contain Key    ${body}    name

Create Post Returns 201
    Create Session    myapi    ${BASE_URL}

    # ข้อมูลที่จะส่งไปสร้างใหม่
    ${payload}=    Create Dictionary
    ...    title=test post
    ...    body=hello world
    ...    userId=1

    # ส่ง POST request
    ${response}=    POST On Session    myapi    /posts    json=${payload}

    # เช็ค status code
    Should Be Equal As Numbers    ${response.status_code}    201
3 รัน test 
robot robot_tests/api_test.robot

⚖️ เปรียบเทียบ Robot vs Playwright

หัวข้อRobot FrameworkPlaywright (Python)
โครงสร้างทุกอย่างในไฟล์เดียว 4 ส่วนแยกหลายไฟล์ pages/ และ tests/
ภาษาที่ใช้keyword ภาษาธรรมดาPython
ความยากในการอ่าน✅ อ่านง่าย ไม่ต้องรู้ codeต้องรู้ Python
เหมาะกับใครQA ที่ไม่ถนัด codeQA ที่ถนัด code
Report✅ สวยงามในตัวต้องติดตั้ง plugin เพิ่ม
ความยืดหยุ่นจำกัดกว่า✅ ยืดหยุ่นกว่ามาก
CI/CD✅ รองรับ✅ รองรับ
ทำ API test ได้มั้ย✅ ได้ ผ่าน RequestsLibrary✅ ได้ ผ่าน page.request

❓ คำถามสัมภาษณ์ที่ควรรู้

ระดับเริ่มต้น

Q: Robot Framework คืออะไร?
A: Tool สำหรับเขียน automated test โดยใช้ keyword-driven ทำให้อ่านง่าย ไม่ต้องรู้ code ลึก

Q: โครงสร้าง Robot Framework มีกี่ส่วน อะไรบ้าง?
A: 4 ส่วน คือ Settings (import library), Variables (ตัวแปร), Test Cases (test จริงๆ), Keywords (keyword ที่สร้างเอง)

Q: Library ใน Robot Framework คืออะไร?
A: ชุดของ keyword สำเร็จรูปที่เรียกใช้ได้เลย เช่น Browser Library มี Click, Fill Text, Get Url ให้ใช้

Q: ${EMPTY} คืออะไร?
A: ตัวแปรพิเศษที่มาพร้อม Robot หมายถึงค่าว่าง "" ใช้ทดสอบกรณีไม่กรอกข้อมูล

ระดับกลาง

Q: Keyword-driven testing คืออะไร?
A: เขียน test เป็น keyword ที่อ่านเข้าใจง่าย ไม่ต้องรู้ code ลึก เหมาะกับ QA ที่ไม่ถนัด programming

Q: Robot Framework ต่างจาก Playwright ยังไง?
A: Robot ทุกอย่างอยู่ในไฟล์เดียว อ่านง่ายกว่า เหมาะ non-dev, Playwright แยกหลายไฟล์ ยืดหยุ่นกว่า เหมาะ dev

Q: Robot Framework ทำ API test ได้มั้ย?
A: ได้ โดยใช้ RequestsLibrary ซึ่งมี keyword เช่น Create Session, GET On Session, POST On Session

Q: report.html กับ log.html ต่างกันยังไง?
A: report.html = สรุปผล pass/fail ภาพรวม, log.html = รายละเอียดทุก step ที่รัน พร้อม screenshot
