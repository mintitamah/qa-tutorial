ภาพรวม Tool ทั้งหมด

Toolทำอะไร
Pythonภาษาที่ใช้เขียน script ทั้งหมด
Playwrightหุ่นยนต์ที่ควบคุม browser แทนเรา
pytestตัวรัน test และสรุปผล pass/fail
requestsตัวยิง API เหมือน Postman แต่เป็น code
Robot Frameworkอีก tool สำหรับเขียน test อ่านง่ายกว่า
GitHubที่เก็บ code บน internet
GitHub Actionsพนักงานอัตโนมัติที่รัน test ให้ทุกครั้งที่ push code

โครงสร้างแต่ละ Tool

Toolโครงสร้าง
API TestImport + Test Cases
PlaywrightImport + Page Object + Test Cases
Robot FrameworkSettings + Variables + Test Cases + Keywords
CI/CDTrigger + Jobs + Steps

Flow การทำงานทั้งหมด

เขียน test code
        ↓
git push ขึ้น GitHub
        ↓
GitHub Actions รัน test อัตโนมัติ
        ↓
ถ้า pass ✅ → merge ได้
ถ้า fail ❌ → แจ้งเตือน ต้องแก้ก่อน
        ↓
ดู report ว่า fail ตรงไหน

คำถามสัมภาษณ์สรุปทั้งหมด

คำถามคำตอบ
Assert คืออะไร?การตรวจสอบว่าผลลัพธ์ตรงกับที่คาดหวังมั้ย ถ้าไม่ตรง test fail ทันที
Page Object Model คืออะไร?Pattern แยก locator ออกเป็น class แยก ทำให้ maintain ง่าย แก้ที่เดียวทุก test ได้รับผล
Playwright ต่างจาก Selenium?Auto-wait, เร็วกว่า, API ใช้งานง่ายกว่า
Keyword-driven testing คืออะไร?เขียน test เป็น keyword อ่านง่าย ไม่ต้องรู้ code ลึก
CI/CD คืออะไร?ระบบรัน test อัตโนมัติทุกครั้งที่ push code ป้องกัน bug หลุดไป production
Status Code 200, 201, 404, 500?200=สำเร็จ, 201=สร้างสำเร็จ, 404=ไม่เจอ, 500=server พัง
