👣 บทที่ 4b: จับมือทำ CI/CD บน Azure DevOps ทีละขั้นตอน

สิ่งที่ต้องมีก่อน:
✅ มี account Azure DevOps (สมัครฟรีที่ dev.azure.com)

✅ มี Python ติดตั้งแล้ว
✅ มี test script เขียนไว้แล้ว (จากบทที่ 1-3)

1 สร้าง Project ใน Azure DevOps

Project คือ "พื้นที่ทำงาน" ที่รวม Repo, Pipeline, Boards ไว้ด้วยกัน

1. เปิด https://dev.azure.com
2. Login ด้วย account Microsoft
3. กด "New Organization" ถ้ายังไม่มี หรือเลือก Organization ที่มีอยู่
4. กด "New Project"
5. ตั้งชื่อ Project เช่น "QA-Automation"
6. เลือก Visibility: Private
7. กด "Create Project"
✅ ได้ Project ใหม่แล้ว จะเห็นหน้า Project Overview

2 สร้าง Repo ใน Azure DevOps

Repo คือที่เก็บ code ของเรา เหมือน Google Drive แต่สำหรับ code

1. ใน Project ที่สร้าง กดที่ "Repos" ในเมนูซ้าย
2. จะเห็น Repo ที่ชื่อเดียวกับ Project ถูกสร้างให้อัตโนมัติ
3. กด "Initialize" เพื่อสร้าง branch main
4. Copy URL ของ Repo ไว้ใช้ในขั้นตอนถัดไป
   ตัวอย่าง: https://dev.azure.com/your-org/QA-Automation/_git/QA-Automation
✅ ได้ Repo แล้ว พร้อมเก็บ code

3 เตรียม code บนเครื่องเรา

เปิด Terminal แล้วสร้างโครงสร้างโปรเจกต์

# สร้างโฟลเดอร์โปรเจกต์
mkdir qa-automation
cd qa-automation

# สร้าง virtual environment
python3 -m venv venv
source venv/bin/activate   # Mac

# ติดตั้ง library
pip install pytest playwright pytest-playwright requests robotframework robotframework-browser

# ติดตั้ง browser
playwright install chromium

# สร้างโครงสร้างโฟลเดอร์
mkdir pages tests api_tests robot_tests

4 สร้างไฟล์ azure-pipelines.yml

ไฟล์นี้ต้องอยู่ใน root ของโปรเจกต์ (ระดับเดียวกับ requirements.txt)

# azure-pipelines.yml
# ไฟล์นี้บอก Azure DevOps ว่าให้ทำอะไรเมื่อมี code ใหม่เข้ามา

# trigger = รัน pipeline เมื่อไหร่
# ทุกครั้งที่ push ขึ้น branch main หรือ develop
trigger:
  branches:
    include:
      - main
      - develop

# pool = รันบนเครื่องอะไร
# ubuntu-latest = เครื่อง Linux ที่ Azure เตรียมให้ฟรี
pool:
  vmImage: ubuntu-latest

# stages = ขั้นตอนใหญ่ๆ แต่ละ stage รันแยกกัน
stages:

# ════════════════════════════
# Stage 1: Playwright UI Tests
# ════════════════════════════
- stage: PlaywrightTests
  displayName: 'Playwright UI Tests'
  jobs:
  - job: RunPlaywright
    displayName: 'Run Playwright Tests'
    steps:

    # ขั้นตอนที่ 1: ติดตั้ง Python
    - task: UsePythonVersion@0
      inputs:
        versionSpec: '3.11'
      displayName: 'Setup Python 3.11'

    # ขั้นตอนที่ 2: ติดตั้ง library
    - script: |
        pip install -r requirements.txt
        playwright install chromium
      displayName: 'Install dependencies'

    # ขั้นตอนที่ 3: รัน test
    - script: |
        pytest tests/ -v --html=playwright-report.html
      displayName: 'Run Playwright tests'

    # ขั้นตอนที่ 4: เก็บ report ไว้ให้ดาวน์โหลด
    - task: PublishBuildArtifacts@1
      condition: always()
      inputs:
        pathToPublish: 'playwright-report.html'
        artifactName: 'playwright-report'
      displayName: 'Publish Playwright Report'

# ════════════════════════════
# Stage 2: API Tests
# ════════════════════════════
- stage: APITests
  displayName: 'API Tests'
  dependsOn: PlaywrightTests   # รอให้ Stage 1 เสร็จก่อน
  jobs:
  - job: RunAPITests
    steps:
    - task: UsePythonVersion@0
      inputs:
        versionSpec: '3.11'
    - script: pip install -r requirements.txt
      displayName: 'Install dependencies'
    - script: pytest api_tests/ -v
      displayName: 'Run API tests'

# ════════════════════════════
# Stage 3: Robot Framework Tests
# ════════════════════════════
- stage: RobotTests
  displayName: 'Robot Framework Tests'
  dependsOn: APITests   # รอให้ Stage 2 เสร็จก่อน
  jobs:
  - job: RunRobotTests
    steps:
    - task: UsePythonVersion@0
      inputs:
        versionSpec: '3.11'
    - script: |
        pip install -r requirements.txt
        rfbrowser init
      displayName: 'Install dependencies'
    - script: robot robot_tests/
      displayName: 'Run Robot tests'
    - task: PublishBuildArtifacts@1
      condition: always()
      inputs:
        pathToPublish: 'report.html'
        artifactName: 'robot-report'
      displayName: 'Publish Robot Report'
⚠️ สำคัญมาก! ไฟล์นี้ต้องชื่อ azure-pipelines.yml เท่านั้น
และต้องอยู่ใน root ของโปรเจกต์ ไม่ใช่ในโฟลเดอร์ย่อย

5 Push code ขึ้น Azure Repos

เปิด Terminal แล้วพิมพ์ทีละบรรทัด

# เข้าไปในโฟลเดอร์โปรเจกต์
cd qa-automation

# เริ่มต้นใช้ git
git init

# บอก git ว่าจะ track ไฟล์ทั้งหมด
git add .

# บันทึก snapshot พร้อมข้อความอธิบาย
git commit -m "feat: add QA automation tests and pipeline"

# เชื่อม local กับ Azure Repos
# แทนที่ URL ด้วย URL ที่ copy มาจากขั้นตอนที่ 2
git remote add origin https://dev.azure.com/your-org/QA-Automation/_git/QA-Automation

# เปลี่ยนชื่อ branch เป็น main
git branch -M main

# push code ขึ้น Azure Repos
git push -u origin main
✅ code ขึ้น Azure Repos แล้ว ไปดูใน Repos จะเห็นไฟล์ทั้งหมด

6 สร้าง Pipeline ใน Azure DevOps

1. ใน Project กดที่ "Pipelines" ในเมนูซ้าย
2. กด "New Pipeline"
3. เลือก "Azure Repos Git"
4. เลือก Repo ที่เพิ่ง push ขึ้นไป
5. เลือก "Existing Azure Pipelines YAML file"
6. เลือก Branch: main
7. เลือก Path: /azure-pipelines.yml
8. กด "Continue"
9. ดูไฟล์ที่แสดงขึ้นมา แล้วกด "Run"
✅ Pipeline ถูกสร้างแล้ว และกำลังรันครั้งแรก!

7 ดู Pipeline รัน

1. ไปที่ Pipelines → เลือก pipeline ที่สร้าง
2. จะเห็น stages ทั้งหมด:
   - Playwright UI Tests
   - API Tests
   - Robot Framework Tests
3. คลิกเข้าไปแต่ละ stage เพื่อดู log
4. ถ้า test ผ่าน → แสดง ✅ สีเขียว
5. ถ้า test fail → แสดง ❌ สีแดง พร้อม error message
6. ดาวน์โหลด report ได้จาก Artifacts ด้านขวา
✅ เสร็จแล้ว! ตอนนี้ทุกครั้งที่ push code ขึ้น Azure Repos
Pipeline จะรัน test อัตโนมัติทันที ไม่ต้องสั่งเอง

🔁 ทุกครั้งที่แก้ code หรือเพิ่ม test ใหม่

# แก้ code หรือเพิ่ม test ใหม่ใน VS Code
# แล้วทำแค่นี้

git add .
git commit -m "feat: เพิ่ม test case login"
git push

# Pipeline จะรัน test อัตโนมัติทันที ไม่ต้องทำอะไรเพิ่ม

🔍 ถ้า Pipeline Fail ต้องทำอะไร?

วิธีดู Error

1. ไปที่ Pipelines → คลิก pipeline run ที่ fail
2. คลิก stage ที่แสดง ❌
3. คลิก job ที่ fail
4. ดู log ที่แสดงขึ้นมา
5. หา error message ว่า fail ตรงไหน เพราะอะไร

ตัวอย่าง Error ที่เจอบ่อย

Errorสาเหตุวิธีแก้
ModuleNotFoundErrorลืมใส่ library ใน requirements.txtเพิ่ม library ใน requirements.txt แล้ว push ใหม่
playwright install failedลืมรัน playwright install chromiumเพิ่ม playwright install chromium ใน pipeline
Test FAILEDtest case fail จริงๆแก้ code หรือแก้ test แล้ว push ใหม่
YAML syntax errorเขียน azure-pipelines.yml ผิดเช็ค indentation ใน yml ให้ถูกต้อง

📋 สรุปทั้งหมด

สร้าง Project ใน Azure DevOps
        ↓
สร้าง Repo
        ↓
เตรียม code + requirements.txt + azure-pipelines.yml
        ↓
git push ขึ้น Azure Repos
        ↓
สร้าง Pipeline โดยชี้ไปที่ azure-pipelines.yml
        ↓
Pipeline รัน test อัตโนมัติทุกครั้งที่ push code
        ↓
ดูผลและ report ใน Azure DevOps
💡 จำไว้:
azure-pipelines.yml = สูตรที่บอก Pipeline ว่าให้ทำอะไร
Repo = ที่เก็บ code และไฟล์ yml
Pipeline = ตัวที่อ่าน yml แล้วทำตาม
