🤔 CI/CD คืออะไรกันแน่?

คำย่อมาจากทำอะไรเปรียบเหมือน
CIContinuous Integrationรัน test อัตโนมัติทุกครั้งที่ push codeQC ตรวจสอบชิ้นส่วนทุกชิ้นที่เข้ามา
CDContinuous Deliveryส่ง code ไป deploy อัตโนมัติหลัง test ผ่านส่งรถที่ผ่าน QC ออกไปขายเลย
ในฐานะ QA เราสนใจแค่ CI เป็นหลัก
คือการรัน test อัตโนมัติทุกครั้งที่มีการ push code ขึ้นมา

🔧 Pipeline คืออะไร? เกี่ยวกับ CI/CD ยังไง?

Pipeline คือ "สูตรการทำงาน" ที่เราเขียนไว้ล่วงหน้า
บอกว่าเมื่อมี code ใหม่เข้ามา ให้ทำอะไรบ้าง เรียงตามลำดับ

CI/CD คือแนวคิด
Pipeline คือวิธีที่ทำให้ CI/CD เกิดขึ้นจริง 
CI/CD = แนวคิดว่า "อยากให้รัน test อัตโนมัติ"
Pipeline = ไฟล์ที่เขียนว่า "ให้ทำแบบนี้นะ"

เหมือนกับ:
CI/CD = อยากได้รถยนต์คุณภาพดี (แนวคิด)
Pipeline = สายพานการผลิตที่ออกแบบไว้ (วิธีทำ)

🏗️ Azure DevOps มีอะไรบ้าง?

Azure DevOps คือ platform ของ Microsoft ที่รวมทุกอย่างไว้ในที่เดียว

ส่วนคืออะไรเปรียบเหมือน
Reposที่เก็บ code ทั้งหมดGoogle Drive สำหรับ code
Pipelinesรัน test และ deploy อัตโนมัติสายพานโรงงาน
Boardsจัดการงาน task และ bugJira หรือ Trello
Artifactsเก็บผลลัพธ์ เช่น reportที่เก็บของที่ผลิตได้
Test Plansจัดการ test caseสมุดบันทึก test case
Azure DevOps
│
├── Repos       ← เก็บ code ของเรา (script test ทั้งหมด)
│
├── Pipelines   ← อ่านไฟล์ azure-pipelines.yml แล้วรันตาม
│
└── Artifacts   ← เก็บ report ที่ได้หลัง test รันเสร็จ

📄 azure-pipelines.yml คืออะไร? ทำไมต้องมี?

azure-pipelines.yml คือไฟล์ที่บอก Azure DevOps ว่า:
1. รัน pipeline เมื่อไหร่? (เช่น ทุกครั้งที่ push ขึ้น branch main)
2. รันบนเครื่องอะไร? (เช่น Linux, Windows)
3. ทำอะไรบ้าง? (เช่น ติดตั้ง Python → รัน test → เก็บ report)

ไฟล์นี้ต้องอยู่ใน Repo เดียวกับ code ของเรา 
# azure-pipelines.yml — อ่านทีละบรรทัด

# trigger = รัน pipeline เมื่อไหร่
trigger:
  branches:
    include:
      - main      # ทุกครั้งที่ push ขึ้น branch main

# pool = รันบนเครื่องอะไร
pool:
  vmImage: ubuntu-latest   # เครื่อง Linux ที่ Azure เตรียมให้ฟรี

# steps = ทำอะไรบ้าง เรียงตามลำดับ
steps:

# ขั้นตอนที่ 1: ติดตั้ง Python
- task: UsePythonVersion@0
  inputs:
    versionSpec: '3.11'
  displayName: 'ติดตั้ง Python 3.11'

# ขั้นตอนที่ 2: ติดตั้ง library
- script: |
    pip install -r requirements.txt
    playwright install chromium
  displayName: 'ติดตั้ง library'

# ขั้นตอนที่ 3: รัน test
- script: pytest tests/ -v
  displayName: 'รัน Playwright tests'

📦 Repo คืออะไร? ทำไมต้องเซต?

คำศัพท์ความหมายเปรียบเหมือน
Repository (Repo)ที่เก็บ code ทั้งหมดบน cloudGoogle Drive สำหรับ code
Branchสาขาของ codeสำเนาของ code ที่แยกออกมา
Commitบันทึก snapshot ของ codeกด Save พร้อมเขียนว่าแก้อะไร
Pushส่ง code จากเครื่องเราขึ้น Repoอัปโหลดขึ้น cloud
Pull Request (PR)ขอ merge code จาก branch นึงไปอีก branchส่งงานให้หัวหน้าตรวจก่อน merge
ทำไมต้องเซต Repo?
เพราะ Pipeline ต้องรู้ว่า code อยู่ที่ไหน ถึงจะไปดึง code มารัน test ได้
ถ้าไม่มี Repo → Pipeline ไม่รู้จะเอา code มาจากไหน → รัน test ไม่ได้

ระดับเริ่มต้น

Q: CI/CD คืออะไร?
A: แนวคิดที่ทำให้การรัน test และ deploy เป็นอัตโนมัติ ทุกครั้งที่มีการ push code ไม่ต้องสั่งเอง

Q: Pipeline คืออะไร?
A: ไฟล์ที่เขียนสูตรการทำงานไว้ล่วงหน้า บอกว่าเมื่อมี code ใหม่เข้ามาให้ทำอะไรบ้าง เรียงตามลำดับ

Q: CI/CD กับ Pipeline เกี่ยวกันยังไง?
A: CI/CD คือแนวคิด Pipeline คือวิธีที่ทำให้แนวคิดนั้นเกิดขึ้นจริง เหมือน CI/CD คือความต้องการ Pipeline คือการลงมือทำ

Q: azure-pipelines.yml คืออะไร?
A: ไฟล์ที่บอก Azure DevOps ว่าให้รัน pipeline เมื่อไหร่ บนเครื่องอะไร และทำอะไรบ้าง

ระดับกลาง

Q: trigger ใน pipeline คืออะไร?
A: บอกว่าให้รัน pipeline เมื่อไหร่ เช่น ทุกครั้งที่ push ขึ้น branch main หรือเมื่อมี Pull Request

Q: ทำไมต้องมี Repo ก่อนถึงจะตั้ง Pipeline ได้?
A: เพราะ Pipeline ต้องดึง code จาก Repo มารัน ถ้าไม่มี Repo Pipeline ไม่รู้จะเอา code มาจากไหน

Q: ถ้า test fail ใน pipeline ต้องทำอะไร?
A: ดู error log ใน pipeline แก้ code แล้ว push ขึ้นใหม่ pipeline จะรันอีกครั้งอัตโนมัติ

Q: Artifact ใน pipeline คืออะไร?
A: ผลลัพธ์ที่ได้จาก pipeline เช่น test report ที่ดาวน์โหลดมาดูได้หลัง pipeline รันเสร็จ

📋 สรุปภาพรวม

CI/CD = แนวคิด "รัน test อัตโนมัติทุกครั้งที่ push code"
        ↓
Pipeline = วิธีทำให้เกิดขึ้นจริง โดยเขียนใน azure-pipelines.yml
        ↓
Repo = ที่เก็บ code และไฟล์ azure-pipelines.yml
        ↓
Azure DevOps = platform ที่รวม Repo + Pipeline ไว้ด้วยกัน
